Onalign이라는 퍼실리테이션 웹앱의 UI 프로토타입을 만들어줘. 팀이 아이디어 회의부터 회고까지 하나의 프로젝트 안에서 이어서 진행하는 도구야.

# 디자인 시스템
- 톤: 깔끔하지만 딱딱하지 않게. 손으로 붙인 것 같은 포스트잇의 친근함과, 데이터/우선순위를 다루는 신뢰감이 공존해야 함
- 컬러 팔레트(참여자 구분용, 6색 파스텔): pink #f7d3de, olive #dde3ba, blue #bcd9ee, purple #d6c9ee, tan #eecd9c, teal #a9e6d3
- 기본 배경은 화이트/오프화이트, 강조 텍스트나 CTA는 짙은 차콜(#242322) 계열
- 포스트잇: 가로 고정 크기, 세로는 내용에 따라 자동으로 늘어남 (기울기 없이 반듯하게)
- 전체적으로 여백 있는 편이 아니라, 정보 밀도가 있는 실무 도구 느낌으로

# 만들어야 할 화면 (아래 순서대로, 하나의 플로우로 이어지게)

## 1. 랜딩 페이지
- 히어로: 헤드라인 "흩어진 의견을, 하나의 방향으로" / 서브헤드 "그 얘기, 저번 회의에서도 하지 않았나요? 결론 없이 길어지는 회의와, 말할 타이밍을 놓친 팀원을 위해 만들었습니다"
- 주 CTA 버튼 "프로젝트 시작하기" + 작은 보조문구 "팀원으로 초대받으셨나요? 공유받은 링크로 바로 들어가시면 됩니다"
- 문제 제기 섹션 "이 두 장면, 낯설지 않으실 거예요": 카드 2개
  - 카드1(제자리걸음): "한 시간 넘게 얘기했는데, 저번 회의랑 결론이 똑같다" + 순환 화살표 아이콘 + "연 392시간, 67% 비생산적으로 느낌" 통계
  - 카드2(소극적 참여): "회의 내내 한마디도 못 하고 끝난 사람, 사실 할 말은 있었을 겁니다" + 말풍선 크기 대비 일러스트("지금까지"는 한 명만 큰 말풍선, "Onalign에서는"은 전원 동일 크기)
- 근거 카드 2개(Production Blocking, NGT) + 하단에 "디지털 환경·소규모 팀 효과는 검증 중" 단서 문구
- 4단계 흐름도(의견 작성→병합→문제 정리→우선순위 투표)를 화살표로 연결
- 차별점 카드 3개: "산으로 가지 않는 회의" / "말이 느린 사람도, 늦지 않습니다" / "시작과 끝을 잇다"
- 마무리 CTA

## 2. 프로젝트 목록 화면
- 상단 고정 로고(클릭 시 이 화면으로), "내 프로젝트" 표시
- "+ 새 프로젝트" 입력(이름 필수 + 목표 한 줄 선택)
- 프로젝트 카드 리스트: 제목, 생성일, 고정(📌) 토글, 삭제 버튼
- 고정된 프로젝트는 리스트 상단에 노출

## 3. 이름 입력(참여) 화면
- 심플한 중앙 정렬: 프로젝트명 표시 + 이름/닉네임 입력창 + "참여하기" 버튼
- 안내 문구: 이름 입력 시 색상이 자동으로 배정된다는 설명

## 4. 의견 작성 탭 (핵심 화면)
- 상단: 어두운 배경의 안내 문구 배너(퍼실리테이션 설명) + 프로젝트 목표 한 줄이 있으면 "🎯 목표: ..." 형태로 그 아래 고정 표시
- 참여자 색상 배지 나열(이름 + 각자 색상)
- 의견 보드가 여러 개일 수 있음: 보드마다 제목 + "+ 포스트잇" 버튼
- 포스트잇: 가로 고정폭, 세로 가변, 작성자 이름 표시, 우측 상단에 "문제로"/"보류" 작은 버튼
- "문제로" 표시된 포스트잇은 그 보드 맨 위 별도 구획에 고정되어 보임 (일반 포스트잇과 시각적으로 구분)
- 화면 하단에 접이식 "보류함" 섹션 (펼치면 보류된 포스트잇 목록, 각 항목에 "의견으로 되돌리기" 버튼)
- 상단 어딘가에 "🔴 녹음 시작" 버튼(참여자 누구나 누를 수 있음, 누르면 "🔴 녹음 중" 배지가 화면 상단에 항상 보이게 전환)
- "병합 모드" 토글 버튼

## 5. 문제 정리 탭
- 여러 보드에 흩어진 "문제로" 표시 포스트잇을 한곳에 모아 리스트로
- 각 항목: 원본 보드명 표시, 문구 인라인 편집, 부가 설명 입력란

## 6. 우선순위별 결과 탭
- 문제 항목을 득표순으로 정렬한 읽기 전용 리스트, 1위는 강조 스타일
- 투표권 설정(1인당 개수) 컨트롤
- 항목별 투표자 아바타(색상 점) + 득표수

## 7. 문서 탭
- "과정 문서" / "결과 문서" 토글
- 표 형식 미리보기 + "이미지로 저장" / "HTML로 다운로드" / "마크다운으로 다운로드" 버튼

각 화면은 같은 상단 헤더(로고, 내 프로젝트, 이미지로 저장, 참여자 정보)를 공유하는 하나의 앱으로 이어지게 만들어줘. 반응형은 데스크톱 우선으로 하되 모바일에서도 깨지지 않게.
