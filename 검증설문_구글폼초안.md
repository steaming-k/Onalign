# 퍼실리테이션 보드 검증 설문 (Google Forms용 초안)

이 설문의 목적은 의견 수렴이 아니라 반증이다. 즉 "다들 좋다고 하면 좋은 아이디어"가 아니라, "가설 중 어떤 것이 틀렸다는 신호가 나오는가"를 찾는 데 초점을 둔다. 그래서 만족도를 묻는 질문보다, 실제 행동(얼마나 자주, 어떤 상황에서, 어디까지 해봤는지)을 묻는 질문 비중을 높였다. 예외 사항을 따로 묻는 섹션(8번)을 둔 것도 같은 이유다 — 가설에 들어맞지 않는 응답이야말로 설계를 고치게 만드는 정보이기 때문이다.

Google Forms에 그대로 옮길 수 있도록 문항, 유형, 옵션을 실제 폼 구성 순서대로 적었다. 괄호 안 `[검증: ○○]` 표시는 폼에는 넣지 않고, 본인이 결과를 해석할 때 참고할 매핑이다.

---

## 시작 페이지 (설문 설명)

**제목**: 아이디어 회의·회고 방식에 대한 설문 (5분 소요)

**설명**:
> 안녕하세요. 팀 단위로 아이디어 회의나 회고를 진행할 때 실제로 어떤 어려움이 있는지, 그리고 어떤 방식이 도움이 될지 확인하기 위한 설문입니다. 정답은 없으니 실제 경험하신 그대로 답변해 주시면 됩니다. 답변은 서비스 개선 목적으로만 사용되며, 개인 식별 정보는 마지막 문항(선택)에서만 수집합니다.

## 섹션 1. 응답자 프로필 (스크리닝)

**1-1. 최근 6개월 이내에 팀 단위로 아이디어 회의(브레인스토밍, 문제 정의, 기획 논의 등)를 해본 적이 있나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 있다 / 없다
- *"없다" 선택 시 섹션 8(예외 사항)로 바로 이동하도록 분기 설정 [검증: 대상자 필터링]*

**1-2. 그 팀은 주로 어떤 성격이었나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 캠프/부트캠프 프로젝트팀 · 학교 조별 과제 · 회사 업무팀(스타트업) · 회사 업무팀(중견/대기업) · 동아리·사이드 프로젝트 · 기타(단답)
- *[검증: 목표 사용자가 "캠프형 팀"에 실제로 집중돼 있는지, 아니면 다른 유형에서도 같은 문제가 나오는지]*

**1-3. 그 팀의 인원 수는?**
- 유형: 객관식(라디오), 필수
- 옵션: 2~4명 / 5~8명 / 9~15명 / 16명 이상
- *[검증: 목표 사용자 규모(5~15명) 및 동시 접속 상한 가설]*

**1-4. 그 팀에서 아이디어 회의는 보통 얼마나 자주 있었나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 거의 매일 / 주 1~2회 / 격주~월 1회 / 프로젝트 시작 시 1회뿐 / 비정기적

## 섹션 2. 문제 정의 검증

**2-1. 아이디어 회의에서 아래 중 가장 많은 시간을 쓴 단계는?**
- 유형: 객관식(라디오), 필수
- 옵션: 문제 현상 파악 및 정의 / 해결 아이디어 브레인스토밍 / 아이디어 중 우선순위 결정 / 역할 분담 및 실행 계획 / 기타(단답)
- *[검증: 1장 문제 정의 — "문제 정의 단계에 가장 많은 시간을 쓴다"는 관찰이 본인만의 경험인지 일반적인지]*

**2-2. "서로 다른 말로 비슷한 문제를 얘기하고 있었는데, 그걸 한참 뒤에야 깨달은" 경험이 있나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 자주 있다 / 가끔 있다 / 거의 없다 / 전혀 없다
- *[검증: 가설 F의 전제 — 이 현상 자체가 실재하는지]*

**2-3. (2-2에서 "자주/가끔 있다" 응답자만) 그걸 깨달았을 때, 보통 회의 시간이 얼마나 더 늘어났나요?**
- 유형: 객관식(라디오)
- 옵션: 거의 안 늘어남 / 10분 이내 / 10~30분 / 30분 이상 / 아예 결론 없이 끝남
- *[검증: 문제의 심각도 — 이게 사소한 불편인지 실제로 회의를 망치는 수준인지]*

**2-4. 그런 상황이 반복되는 이유로 가장 가까운 것은?** (복수 선택 가능)
- 유형: 체크박스
- 옵션: 각자 용어를 다르게 씀 / 누가 무슨 의도로 말했는지 물어보기 애매한 분위기 / 회의 중 기록이 안 남아 나중에 헷갈림 / 발언 순서를 기다리다 다른 얘기로 넘어감 / 잘 모르겠다 / 기타(단답)
- *[검증: 문제의 원인이 "의도 파악 실패"에 가까운지, 아니면 다른 원인(예: 발언 기회 부족)이 더 큰지 — 1장의 재정의가 맞는지 교차 확인]*

## 섹션 3. 현재 사용 방식

**3-1. 아이디어 회의를 진행할 때 주로 어떤 방식/도구를 쓰나요?** (복수 선택 가능)
- 유형: 체크박스
- 옵션: 화이트보드나 종이(오프라인) / Miro, FigJam, Mural 등 범용 캔버스 / EasyRetro, TeleRetro 등 회고 전용 도구 / 구글 문서·노션 등 텍스트 문서 / 카카오톡·슬랙 등 채팅 / 그냥 말로만 진행 / 기타(단답)

**3-2. 회고(프로젝트 완료 후 되돌아보는 회의)는 따로 진행하나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 매번 진행한다 / 가끔 진행한다 / 거의 안 한다 / 회고라는 걸 해본 적 없다

**3-3. (3-2에서 "매번/가끔" 응답자만) 그 회고는 아이디어 회의 때 썼던 것과 같은 도구·문서를 이어서 쓰나요, 아니면 완전히 새로 시작하나요?**
- 유형: 객관식(라디오)
- 옵션: 같은 도구·문서를 이어서 씀 / 새로 시작하지만 내용은 참고함 / 완전히 새로 시작, 이전 내용은 거의 안 봄
- *[검증: 가설 D 및 4장의 핵심 차별점("아이데이션-회고 연결") — 이걸 실제로 아무도 안 하고 있다면, 그만큼 이 기능의 체감 가치가 낮을 수 있음]*

**3-4. 프로젝트가 시작해서 끝날 때까지 보통 기간은?**
- 유형: 객관식(라디오), 필수
- 옵션: 1주 이내 / 1~2주 / 3~4주 / 1~3개월 / 3개월 이상 / 프로젝트마다 다름
- *[검증: 9장 리스크 — 아이데이션-회고 간격이 짧아야 연결 기능이 자연스럽다는 가설]*

## 섹션 4. 제안 기능에 대한 반응

*(이 섹션 전에 "이런 도구를 구상 중입니다"라는 1~2문장 소개와 스크린샷을 넣을 것을 권한다 — 텍스트만으로는 "병합", "우선순위 투표" 같은 용어가 와닿지 않을 수 있다.)*

**4-1. 이름만 입력하면 로그인 없이 바로 참여할 수 있는 방식에 대해 어떻게 생각하나요?**
- 유형: 척도(1~5), 필수
- 라벨: 1 = 전혀 필요 없다, 5 = 매우 필요하다
- *[검증: 가설 A]*

**4-2. 비슷한 의견들을 참가자가 직접 골라 하나로 합치는 기능(병합)이 있다면, 회의에 도움이 될 것 같나요?**
- 유형: 척도(1~5), 필수
- *[검증: 가설 F]*

**4-3. 우선순위 투표에서, 한 사람이 같은 항목에 표를 두 개 이상 몰아줄 수 없게 막는 규칙(→ 대신 여러 항목에 나눠 투표)에 대해 어떻게 생각하나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 좋다 — 넓은 동의를 보기 편함 / 나쁘다 — 정말 원하는 항목에 몰아주고 싶음 / 상관없다
- *[검증: 가설 B]*

**4-4. 회의 결과(의견, 문제 목록, 투표 결과)를 이미지 한 장으로 저장해 공유할 수 있다면, 주로 어떤 용도로 쓸 것 같나요?**
- 유형: 체크박스
- 옵션: 팀 내부 공유(슬랙, 노션 등) / 개인 아카이빙(나중에 참고용) / 상사·클라이언트 보고 / 딱히 안 쓸 것 같다
- *[검증: 6장에서 미확정이었던 이미지 내보내기의 실제 목적지]*

**4-5. 회의 주제가 여러 개일 때, 주제별로 보드를 나눠 각각 의견을 받는 기능이 필요할 것 같나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 필요하다, 자주 쓸 것 같다 / 있으면 가끔 쓸 것 같다 / 없어도 상관없다, 한 보드로 충분
- *[검증: 다중 의견 보드 기능의 실제 수요]*

## 섹션 6. 계정 및 로그인 방식

**6-1. 프로젝트를 만든 사람만 구글/깃허브로 로그인하고, 참여하는 팀원은 이름만 입력하는 방식에 대해 어떻게 생각하나요?**
- 유형: 척도(1~5), 필수
- 라벨: 1 = 어색하다/불편하다, 5 = 자연스럽다
- *[검증: 가설 E]*

**6-2. (참여자 입장이라면) 본인이 참여했던 프로젝트 목록을 나중에 따로 확인하고 싶나요?**
- 유형: 객관식(라디오), 필수
- 옵션: 그렇다, 계정이 있었으면 좋겠다 / 아니다, 그 자리에서만 확인되면 충분 / 잘 모르겠다
- *[검증: 역질문 2번 — 참여자 계정이 실제로 필요한 요구인지, 아니면 만든 사람만의 착각인지]*

## Google Forms 설정 팁

- 1-1 문항 뒤에 "섹션 이동" 로직을 걸어, "없다" 선택 시 섹션 8로 바로 이동하게 하면 대상이 아닌 응답자의 이탈을 줄일 수 있다.
- 2-3, 3-3처럼 "이전 답변에 따라만 보이는" 문항은 Google Forms의 "조건부 섹션 이동" 기능으로 처리한다(해당 옵션에서만 다음 섹션으로, 나머지 옵션은 그 다음 섹션으로 건너뛰게 설정).
- 응답 수가 20건을 넘기 전에는 척도형(4-1, 4-2, 5-1, 6-1) 점수의 평균보다 분포(1~2점과 4~5점이 각각 몇 명인지)를 보는 편이 낫다 — 소수 응답에서는 평균이 쉽게 왜곡된다.
